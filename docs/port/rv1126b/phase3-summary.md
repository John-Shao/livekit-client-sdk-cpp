## 关键决策与坑

### (a) 不修改 upstream `client-sdk-rust/.cargo/config.toml`

上游已有 `rustflags = ["-C", "link-arg=-fuse-ld=lld"]`，注释明说 "need lld to link libwebrtc.a"。我们用 env var (`CARGO_TARGET_AARCH64_UNKNOWN_LINUX_GNU_LINKER` 等) 而不是改上游文件，保 submodule 干净。

### (b) `-fuse-ld=lld` 触发的 collect2 cannot find ld

Buildroot toolchain 内置只有 `ld`/`ld.bfd`，没有 `ld.lld`。GCC 的 collect2 在 `-fuse-ld=lld` 下找不到 `ld.lld` 走系统 PATH，而 GCC 的 exec search path 不含 `/usr/bin`。修法：在 toolchain 的 `host/aarch64-buildroot-linux-gnu/bin/` 软链 `ld.lld → /usr/bin/ld.lld`（env-rv1126b.sh 幂等执行）。

### (c) 双 protoc 路径问题

CMakeLists.txt 在 `include(protobuf)` 之后**强制覆盖** `Protobuf_PROTOC_EXECUTABLE = $<TARGET_FILE:protobuf::protoc>`，把我们传入的 host protoc 抢回成 vendored aarch64 binary。改法：CMakeLists.txt 和 protobuf.cmake **两处**都加 `if(CMAKE_CROSSCOMPILING)` 分支，让 host protoc 优先。

### (d) protoc 25.3 vs 3.6.1 版本不兼容

最初想直接用系统 `/usr/bin/protoc 3.6.1`（apt focal 自带）。但 protobuf 25.3 runtime 的 `.pb.h` 自检拒绝 3.6.1 生成的代码：

```
#error This file was generated by an older version of protoc which is
#error incompatible with your Protocol Buffer headers.
```

修法：用 vendored protobuf-25.3 源码在 host 上**本地编一份 25.3 protoc**（同源 ABI 严格匹配）。protobuf 的 tarball 缺 `third_party/abseil-cpp/`（git submodule 在 release tarball 里没带），先 symlink 到我们已下载的 abseil-20240116.2 才能 configure 通过。

### (e) cmake 双重源问题

Buildroot host 自带 cmake 3.28（在 `host/usr/share/cmake-3.28`），但其 libcurl 没编 HTTPS（FetchContent 全挂）。系统 apt cmake 是 3.16.3 < 项目要求 3.20。最终用 `pip install --user cmake` → 4.3.2，把 `~/.local/bin` 前置到 PATH。

### (f) PATH 排序：toolchain 后置

把 `$ATK_TOOLCHAIN_ROOT/usr/bin` 放到 PATH **末尾**而非开头。带前缀的 `aarch64-buildroot-linux-gnu-*` 系统里没有，仍从 toolchain 命中；但 `cmake / make / python3` 等无前缀工具走系统的版（避开 Buildroot 那份没 HTTPS 的 cmake、可能太新的 python 等）。

### (g) libyuv 源码靠 Windows 宿主机 VPN

`yuv-sys/libyuv` git submodule 指向 `chromium.googlesource.com`，国内 GFW 阻断。Windows 这边有 VPN 客户端（端口 15236），从 Windows clone 后 `scp -r` 给 VM。HEAD 落在项目固定的 SHA `917276084a49be726c90292ff0a6b0a3d571a6af` 上。

## 完整重现命令（供新机器参考）

```bash
# (一次性) 装 host 工具
sudo apt install -y protobuf-compiler lld libclang-dev clang
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y --profile minimal
. $HOME/.cargo/env
rustup target add aarch64-unknown-linux-gnu
python3 -m pip install --user cmake

# (一次性) 准备 deps：webrtc prebuilt / libyuv / abseil / protobuf / spdlog（见 §VM 上的状态）
# 以及编 host protoc：
cd ~/lk-deps/protobuf-25.3
ln -s ~/lk-deps/abseil-cpp-20240116.2 third_party/abseil-cpp     # tarball 缺 git submodule
mkdir -p build-host && cd build-host
cmake .. -DCMAKE_BUILD_TYPE=Release -Dprotobuf_BUILD_TESTS=OFF \
  -Dprotobuf_BUILD_EXAMPLES=OFF -Dprotobuf_BUILD_PROTOC_BINARIES=ON \
  -Dprotobuf_ABSL_PROVIDER=module
cmake --build . --target protoc -j$(nproc)

# (每次 build) 进项目 + 编
cd ~/livekit/livekit-sdk-cpp-0.3.3
source scripts/env-rv1126b.sh                                    # 自动设 LK_CUSTOM_WEBRTC、ld.lld 软链等
cmake -S . -B build-rv1126b \
  -DCMAKE_TOOLCHAIN_FILE=cmake/toolchains/rv1126b-aarch64-linux-gnu.cmake \
  -DCMAKE_BUILD_TYPE=Release \
  -DLIVEKIT_BUILD_EXAMPLES=ON \
  -DLIVEKIT_BUILD_TESTS=OFF \
  -DProtobuf_PROTOC_EXECUTABLE=$HOME/lk-deps/protobuf-25.3/build-host/protoc-25.3.0 \
  -DFETCHCONTENT_SOURCE_DIR_LIVEKIT_ABSEIL=$HOME/lk-deps/abseil-cpp-20240116.2 \
  -DFETCHCONTENT_SOURCE_DIR_LIVEKIT_PROTOBUF=$HOME/lk-deps/protobuf-25.3 \
  -DFETCHCONTENT_SOURCE_DIR_LIVEKIT_SPDLOG=$HOME/lk-deps/spdlog-1.15.1
cmake --build build-rv1126b -j$(nproc)
```
